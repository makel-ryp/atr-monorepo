## Prerequisites

App Agent requires:
- **Bun** (v1.0 or later) - [Install Bun](https://bun.sh)
- **Node.js** (v20 or later) - for Nuxt compatibility
- **Git** - for version control

## Quick Start

### 1. Clone or Fork

**For evaluation:**
```bash
git clone https://github.com/app-agent/app-agent.git
cd app-agent
```

**For production use:**
1. Fork the repository on GitHub
2. Clone your fork
3. Keep the upstream remote for updates

```bash
# Clone your fork
git clone https://github.com/YOUR-ORG/app-agent.git
cd app-agent

# Add upstream for updates
git remote add upstream https://github.com/app-agent/app-agent.git
```

### 2. Install Dependencies

```bash
bun install
```

This installs dependencies for all workspaces: core, docs, and demos.

### 3. Start Development

```bash
bun run dev
```

**First-time experience:**

If `/apps/` is empty (it will be on a fresh clone), you'll see an interactive prompt:

```
No apps found in /apps/. What would you like to do?

1. Copy a demo to start your first app
2. Run demos to explore first
3. Exit

Enter choice (1-3):
```

This guides you to either:
- **Copy a demo** → Start building immediately with a working pattern
- **Explore demos** → See what's possible before deciding
- **Exit** → Run `bun run dev:docs` separately to browse documentation

## What Runs Where

| Service | Port | Description |
|---------|------|-------------|
| Documentation | 3000 | This documentation + MCP server |
| Your apps | 3001+ | Customer applications in `/apps/` |
| Dashboard demo | 3010 | Admin panels, internal tools |
| SaaS demo | 3011 | Customer-facing app patterns |
| Landing demo | 3012 | Marketing sites, landing pages |
| Chat demo | 3013 | AI chatbot with persistent history |

## Available Scripts

| Command | Description |
|---------|-------------|
| `bun run dev` | Smart launcher - guides first-time setup or runs your apps |
| `bun run dev:demos` | Run all demo applications |
| `bun run dev:docs` | Run documentation server only |
| `bun run dev:all` | Run everything (docs + demos + apps) |
| `bun run build` | Build your apps for production |
| `bun run typecheck` | Run TypeScript type checking |

## Pulling Upstream Updates

When we release updates to the core layer:

```bash
# Fetch upstream changes
git fetch upstream

# Merge into your branch
git merge upstream/main

# Resolve any conflicts (rare - your code is in /apps/)
# Push to your fork
git push origin main
```

The layering architecture means your customizations in `/apps/` rarely conflict with upstream changes to `/core/` and `/demos/`.
