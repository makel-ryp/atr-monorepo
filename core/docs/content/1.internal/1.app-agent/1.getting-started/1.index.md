## Quick Start

```bash
# Clone the repo
git clone https://github.com/app-agent/app-agent.git
cd app-agent

# Install dependencies
bun install

# Start development
bun run dev
```

If `/apps/` is empty, you'll be prompted to copy a demo as your starting point.

## Architecture

The layering system keeps your code isolated from upstream updates:

```
CORE (maintained upstream)
├── Shared components and composables
├── Common Nuxt UI configuration
└── Dev tooling and documentation
         │
    fork + pull upstream
         │
         ▼
YOUR REPO (your GitHub)
├── Your apps in /apps/
├── Your customizations
└── Your infrastructure
```

Pull upstream updates cleanly. Your `/apps/` code rarely conflicts.

## Project Structure

```
app-agent/
├── core/              # Shared Nuxt layer
│   ├── app/           # Components, composables, layouts
│   ├── docs/          # Documentation infrastructure + MCP
│   └── cli/           # Dev tooling
├── docs/              # Your documentation (this site)
├── demos/             # Reference implementations
│   ├── dashboard/     # Internal tools (port 3010)
│   ├── saas/          # Customer apps (port 3011)
│   └── landing/       # Marketing sites (port 3012)
├── apps/              # Your applications
└── packages/          # Optional shared packages
```

## Why the AI Works

Traditional AI code generators fail because:
- No understanding of project conventions
- Context window fills up, AI contradicts itself
- You spend hours debugging AI spaghetti

App Agent's AI succeeds because:
- Patterns are so clear even basic models follow them
- Conventions enforced by the system itself
- MCP gives AI complete codebase understanding
- Demos provide working examples to copy

**The AI isn't smarter. The codebase is smarter.**
