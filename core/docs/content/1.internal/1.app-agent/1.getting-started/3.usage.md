## Creating Your First App

The easiest way to create your first app is to copy a demo:

```bash
bun run dev
```

If `/apps/` is empty, choose option 1 ("Copy a demo to start your first app"), then:
1. Select which demo to copy (dashboard, saas, or landing)
2. Enter a name for your app (e.g., "my-app", "client-portal")

Your new app will be created at `/apps/your-name/` and will automatically extend the core layer.

### Manual Copy

You can also copy a demo manually:

```bash
# Copy the dashboard demo
cp -r demos/dashboard apps/my-app

# Update package.json
cd apps/my-app
# Change "name" to "@app-agent/my-app"
# Change port if needed (default: 3001)
```

## Project Structure

Each app follows Nuxt 4 conventions:

```
apps/my-app/
├── nuxt.config.ts      # Extends ../../core
├── package.json        # App metadata and scripts
├── app/
│   ├── app.vue         # Root component
│   ├── components/     # App-specific components
│   ├── composables/    # App-specific composables
│   ├── layouts/        # Page layouts
│   └── pages/          # File-based routing
└── server/
    └── api/            # Server API routes
```

### The Core Layer

Your app automatically inherits from `/core/`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['../../core'],
  // Your app-specific config
})
```

This gives you:
- Shared components from `core/app/components/`
- Shared composables from `core/app/composables/`
- Shared layouts from `core/app/layouts/`
- Common Nuxt UI configuration

## Running Your App

```bash
# Run your apps (recommended)
bun run dev

# Run with demos
bun run dev:all

# Run just demos (for reference)
bun run dev:demos
```

When you have apps in `/apps/`, `bun run dev` runs:
- Documentation at `localhost:3000`
- Your apps at `localhost:3001+`

## Adding Components

### App-Specific Components

Create components in your app's `app/components/` directory:

```vue [apps/my-app/app/components/MyWidget.vue]
<template>
  <div class="p-4 border rounded">
    <h3>{{ title }}</h3>
    <slot />
  </div>
</template>

<script setup lang="ts">
defineProps<{
  title: string
}>()
</script>
```

### Using Core Components

Components from `/core/app/components/` are automatically available:

```vue
<template>
  <!-- Use any core component directly -->
  <CoreButton @click="handleClick">Click me</CoreButton>
</template>
```

## Adding Pages

Create pages in `app/pages/` with file-based routing:

```vue [apps/my-app/app/pages/dashboard.vue]
<template>
  <div>
    <h1>Dashboard</h1>
    <!-- Your page content -->
  </div>
</template>
```

Dynamic routes use brackets:
- `pages/users/[id].vue` → `/users/123`
- `pages/[...slug].vue` → Catch-all route

## Adding API Routes

Create server API routes in `server/api/`:

```ts [apps/my-app/server/api/users.ts]
export default defineEventHandler(async (event) => {
  return {
    users: [
      { id: 1, name: 'Alice' },
      { id: 2, name: 'Bob' }
    ]
  }
})
```

Access at `/api/users`.

## Working with Demos

Demos are read-only reference implementations. When you need a pattern:

1. Find the demo that has it (dashboard, saas, or landing)
2. Use the MCP tools or browse the code directly
3. Copy the pattern into your app, adapting as needed

### Comparing to a Demo

Keep a pristine demo copy to compare against:

```bash
# Your working app
apps/my-app/

# Reference copy (optional)
apps/dashboard-reference/
```

## Building for Production

```bash
# Build your apps
bun run build

# Preview production build
bun run preview
```

This builds only apps in `/apps/` - demos are never included in production builds.

## TypeScript

TypeScript checking is done via your IDE (VS Code + Volar recommended). For manual checking:

```bash
bun run typecheck
```
