## Overview

App Agent includes a built-in [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) server that enables AI assistants to understand and work with your codebase. Unlike traditional documentation, AI can:

- **Proactively search** documentation while generating responses
- **Introspect your apps** - see structure, components, and source code
- **Understand patterns** across demos and your custom apps

This is why App Agent's AI doesn't break like Bolt/Lovable/V0 - the AI has complete context.

## Quick Setup

Your MCP server runs at `http://localhost:3000/mcp` when the docs are running.

### Claude Code

```bash
claude mcp add --transport http app-agent-docs http://localhost:3000/mcp
```

### Cursor

Create `.cursor/mcp.json` in your project:

```json
{
  "mcpServers": {
    "app-agent-docs": {
      "type": "http",
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

### VS Code (with GitHub Copilot)

Create `.vscode/mcp.json`:

```json
{
  "servers": {
    "app-agent-docs": {
      "type": "http",
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

```json

    "app-agent-docs": {

  }
}
```

## Available Tools

App Agent provides **6 MCP tools** - 2 for documentation, 4 for codebase introspection.

### Documentation Tools

#### `list-pages`

Lists all documentation pages with titles, paths, and descriptions.

```
AI: "What documentation is available?"
→ Returns all pages in the docs
```

#### `get-page`

Retrieves the full markdown content of a documentation page.

| Parameter | Type | Description |
|-----------|------|-------------|
| `path` | string | Page path (e.g., `/getting-started`) |

```
AI: "Show me the installation guide"
→ Returns full content of /getting-started/installation
```

### Codebase Introspection Tools

#### `list-apps`

Discovers all applications and demos in the monorepo.

```
AI: "What apps are in this project?"
→ Returns:
  - apps: [] (customer apps in /apps/)
  - demos: [dashboard, saas, landing] (reference implementations)
  - Each with: name, packageName, port, description
```

**Example response:**
```json
{
  "apps": [],
  "demos": [
    {
      "name": "dashboard",
      "packageName": "@app-agent/demo-dashboard",
      "type": "demo",
      "port": 3010
    }
  ]
}
```

#### `get-app-structure`

Returns the file tree of any app or demo.

| Parameter | Type | Description |
|-----------|------|-------------|
| `appName` | string | App name (e.g., `dashboard`, `my-app`) |

```
AI: "Show me the structure of the dashboard demo"
→ Returns file tree excluding node_modules, .nuxt, etc.
```

**Example response:**
```
dashboard/
├── app/
│   ├── components/
│   │   ├── AppHeader.vue
│   │   └── DataTable.vue
│   ├── pages/
│   │   ├── index.vue
│   │   └── analytics.vue
│   └── composables/
│       └── useApi.ts
├── server/
│   └── api/
│       └── users.ts
└── nuxt.config.ts
```

#### `list-components`

Lists all components, composables, pages, layouts, and API routes in an app.

| Parameter | Type | Description |
|-----------|------|-------------|
| `appName` | string | App name |
| `type` | string | Filter: `all`, `components`, `composables`, `pages`, `layouts`, `api` |

```
AI: "What components does the dashboard have?"
→ Returns list of all Vue components with paths

AI: "Show me the API routes in dashboard"
→ Returns list of server/api/ endpoints
```

#### `get-file`

Reads the source code of any file in the monorepo.

| Parameter | Type | Description |
|-----------|------|-------------|
| `appName` | string | App name or `core` |
| `filePath` | string | Path within the app |

```
AI: "Show me the dashboard's analytics page"
→ Returns source of demos/dashboard/app/pages/analytics.vue
```

**Security:** Only reads allowed file types (.vue, .ts, .js, .json, .md, etc.) and blocks sensitive paths (node_modules, .env, credentials).

## How AI Uses These Tools

### Example Workflow

**User:** "Add a page similar to the dashboard's analytics page to my app"

**AI workflow:**
1. `list-apps` → Finds dashboard in demos, user's app in apps
2. `get-app-structure("dashboard")` → Sees the file layout
3. `list-components("dashboard", "pages")` → Finds analytics.vue
4. `get-file("dashboard", "app/pages/analytics.vue")` → Reads the source
5. Creates similar page in user's app following the same patterns

### Why This Works

Traditional AI code generators fail because:
- No understanding of project conventions
- Context window fills up with irrelevant code
- AI contradicts itself as complexity grows

App Agent's MCP approach succeeds because:
- AI can explore the codebase systematically
- Patterns are discoverable, not guessed
- Demos provide working examples to follow
- The codebase structure enforces conventions

**The AI isn't smarter. The codebase is smarter.**

## Customization

### Adding Custom Tools

Create tools in `core/docs/server/mcp/tools/`:

```ts
// server/mcp/tools/my-tool.ts
import { z } from 'zod'

export default defineMcpTool({
  description: 'What this tool does',
  parameters: z.object({
    input: z.string().describe('Input parameter')
  }),
  handler: async ({ input }) => {
    // Your logic here
    return {
      content: [{ type: 'text', text: result }]
    }
  }
})
```

### Disabling MCP

If needed, disable in `core/docs/nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  mcp: {
    enabled: false
  }
})
```
