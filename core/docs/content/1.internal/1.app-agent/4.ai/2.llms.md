## Overview

App Agent documentation automatically generates `/llms.txt` and `/llms-full.txt` files that provide structured, AI-readable content about your project. This complements the [MCP server](/internal/app-agent/ai/mcp) by offering a static snapshot of documentation.

::prose-note{to="/llms.txt"}
View the `/llms.txt` file generated for this documentation.
::

## How It Works

The [nuxt-llms](https://github.com/nuxt-content/nuxt-llms) module:

1. Scans all documentation pages in `/content/`
2. Extracts titles, descriptions, and content
3. Generates structured text files optimized for LLM consumption
4. Pre-renders files at build time for static hosting

## MCP vs LLMs.txt

| Feature | MCP Server | LLMs.txt |
|---------|-----------|----------|
| Content | Dynamic, real-time | Static snapshot |
| Scope | Docs + codebase introspection | Docs only |
| Access | Requires docs server running | Works with static files |
| Use case | Active development | CI/CD, static hosting |

**Recommendation**: Use MCP during development for full codebase access. Use LLMs.txt for CI/CD pipelines or when deploying docs statically.

## Configuration

The defaults are configured in `core/docs/nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  llms: {
    domain: 'https://app-agent.io',
    title: 'App Agent Documentation',
    description: 'AI-native Nuxt 4 platform documentation',
    full: {
      title: 'App Agent - Complete Documentation',
      description: 'Full documentation for the App Agent platform'
    }
  }
})
```

## For AI Assistants

When working with App Agent, AI assistants should:

1. **First**: Use MCP tools if the docs server is running (more powerful)
2. **Fallback**: Read `/llms.txt` for documentation overview
3. **Deep dive**: Use `get-file` MCP tool to read actual source code
