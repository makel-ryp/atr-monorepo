## Overview

This section covers how to write documentation. The docs are built with [Nuxt Content](https://content.nuxt.com) and run automatically on `localhost:3000` during development.

## Where to Write Documentation

Write your documentation in the `/docs/` folder at the repository root:

```
/docs/
├── internal/                  # Internal team documentation
│   └── 2.your-team/          # Merges into /internal/ section
│       ├── 0.index.md        # Section landing page
│       └── 1.onboarding.md   # Team guides
│
├── 2.company/                 # Top-level section at /company/
│   ├── 0.index.md
│   └── handbook.md
│
└── 3.api/                     # Another top-level section
    └── reference.md
```

### Routing Rules

| You write to | Appears at URL |
|--------------|----------------|
| `/docs/internal/2.team/guide.md` | `/internal/team/guide` |
| `/docs/2.company/handbook.md` | `/company/handbook` |
| `/docs/3.api/reference.md` | `/api/reference` |

### Why `/docs/` Instead of `/core/docs/content/`

The `/docs/` folder is **yours** - no merge conflicts when pulling upstream updates.

The `/core/docs/content/` folder contains upstream App Agent documentation. You *can* edit it, but you'll get merge conflicts when updating. The App Agent reference docs (`/internal/app-agent/`) should never be edited.

## Documentation Server

The documentation server **always runs** during development on `localhost:3000`. This is intentional:

- **MCP Server**: AI assistants connect to `http://localhost:3000/mcp` for codebase introspection
- **Quick Reference**: Your team can look up documentation while coding
- **Auto-reload**: Changes to markdown files appear instantly

## Creating Pages

Create `.md` files with frontmatter:

```md [/docs/2.guides/1.quickstart.md]
---
title: Quickstart Guide
description: Get started in 5 minutes
---

Your content here...
```

## Numbering Convention

Use numbered prefixes to control navigation order:

```
/docs/
├── 1.guides/             # First section
├── 2.reference/          # Second section
└── 3.api/                # Third section
```

- Numbers are stripped from URLs (`2.guides` → `/guides`)
- `0.index.md` becomes the section landing page
- Files without numbers sort alphabetically after numbered files
