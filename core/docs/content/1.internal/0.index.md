## Adding Your Documentation

Add your documentation to the `/docs/` folder at the repository root:

```
/docs/
├── internal/              # Merges into this section
│   └── 2.your-team/       # Appears alongside App Agent
│       ├── 0.index.md
│       └── guides.md
│
└── 2.company/             # Top-level section at /company/
    ├── 0.index.md
    └── handbook.md
```

### Where to Write

| Location | Result |
|----------|--------|
| `/docs/internal/**` | Merges into `/internal/` (this section) |
| `/docs/**` | Creates top-level sections (e.g., `/company/`) |

### Why `/docs/` Instead of `/core/docs/content/`

The `/docs/` folder is yours - no merge conflicts when pulling upstream updates. The `/core/` folder is upstream-maintained; editing it directly works but creates conflicts on updates.

See [Writing Documentation](/internal/app-agent/documentation) for syntax and components.
