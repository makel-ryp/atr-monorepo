# SaaS Demo

The SaaS demo showcases patterns for building customer-facing applications with authentication, user dashboards, and content management.

## Features

- **User Authentication**: Customer login and registration flows
- **User Dashboard**: Personal workspace and data views
- **Content**: Integration with Nuxt Content for dynamic content
- **Mobile-First**: Responsive design patterns

## Running the Demo

```bash
# Run all demos
bun run dev:demos

# Visit http://localhost:3011
```

## Directory Structure

```
demos/saas/
├── app/
│   ├── components/    # SaaS-specific components
│   ├── composables/   # Auth and data composables
│   ├── layouts/       # App and marketing layouts
│   └── pages/         # Application pages
├── content/           # Nuxt Content files
└── nuxt.config.ts     # Extends core layer
```

## Key Patterns

### API Consumption

This demo consumes APIs from the Dashboard demo, showing how multiple apps can share a single API source.

```typescript
// Configure API base in nuxt.config.ts
runtimeConfig: {
  public: {
    apiBase: 'http://localhost:3010/api'
  }
}
```

### Content Integration

Uses Nuxt Content for managing dynamic pages and blog posts.

### Responsive Design

Mobile-first approach with Tailwind CSS and Nuxt UI components.

## Customizing

To use this demo as a starting point:

1. Copy to `/apps/your-app-name`
2. Update `package.json` with your app name
3. Point `apiBase` to your actual API server
4. Customize pages and components
